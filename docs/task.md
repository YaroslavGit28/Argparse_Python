# Техническое задание

## 1. Описание задачи

Нужно разработать учебную консольную программу на Python, которая демонстрирует работу модуля `argparse` на реальном примере. Программа должна запускаться из командной строки, принимать различные аргументы и подкоманды, выполнять простые операции (приветствие, арифметика, работа с файлами) и наглядно показывать, как устроен разбор аргументов. Код и документация должны быть такими, чтобы по ним можно было не только пользоваться программой, но и изучать сам инструмент `argparse`.

## 2. Цель работы

Цель работы — получить практический опыт проектирования интерфейса командной строки с помощью `argparse` и сформировать инженерное понимание того, как модуль помогает структурировать входные данные. В результате должна получиться небольшая, но полноценная CLI‑утилита, которая:

- использует ключевые возможности `argparse`;
- имеет понятный `--help` и примеры запуска;
- сопровождается подробной инженерной документацией.

## 3. Функциональные требования

1. Программа должна быть написана на Python и использовать модуль `argparse` как основной инструмент работы с аргументами командной строки.  
2. Должно быть не менее трёх подкоманд, например:
   - `greet` — выводит одно или несколько приветствий;
   - `calc` — выполняет простые арифметические операции над списком чисел;
   - `file stats` — показывает базовую статистику по текстовому файлу (строки, слова, символы).
3. Для каждой подкоманды должны быть продемонстрированы разные типы аргументов:
   - позиционные;
   - опциональные флаги с булевым значением;
   - аргументы с типами (`int`, `float`, `Path`);
   - аргументы с ограничениями по значениям (`choices`);
   - аргументы с несколькими значениями (`nargs`).
4. Должны использоваться подкоманды (`add_subparsers`) и хотя бы одна взаимно исключающая группа аргументов (`add_mutually_exclusive_group`).  
5. Программа должна поддерживать чтение аргументов из файла через `fromfile_prefix_chars`, чтобы можно было запускать её с `@args.txt`.  
6. Результаты работы должны оформляться в читаемом виде (блоки, заголовки, пояснения), чтобы пользователь ясно понимал, что произошло.

## 4. Нефункциональные требования

1. **Понятность интерфейса**: все подкоманды и аргументы должны иметь осмысленные имена и внятные описания в `--help`.  
2. **Читаемость кода**: структура программы должна быть разделена на функции (построение парсера, обработчики подкоманд, вывод результатов), без “спагетти‑кода” в одном большом блоке.  
3. **Расширяемость**: архитектура должна позволять относительно легко добавлять новые подкоманды и аргументы без полного переписывания программы.  
4. **Портируемость**: программа должна работать в стандартной среде Python 3.x без внешних зависимостей (кроме стандартной библиотеки).  
5. **Документированность**: проект должен содержать README и файлы в `docs/` с инженерным описанием инструмента и практической реализации.

## 5. Ограничения и допущения

1. Не требуется сложная бизнес‑логика: операции могут быть простыми (сложение чисел, подсчёт строк и слов в файле).  
2. Не требуется графический интерфейс: вся работа ведётся только через командную строку.  
3. Обработка ошибок за пределами `argparse` может быть базовой, главное — корректно демонстрировать типовые сценарии валидации входных параметров.  
4. Предполагается запуск на Windows / Linux / macOS при наличии установленного интерпретатора Python 3.x.  
5. В рамках работы допускается, что программа ориентирована на одного пользователя (нет требований по многопользовательскому доступу, безопасности и т.д.).

## 6. Источник ТЗ

Техническое задание сформулировано на основе учебного задания и доработано с помощью нейросети для уточнения формулировок и структуры.
